# 项目结构整理建议与约定

本文档用于统一当前仓库的目录职责、脚本入口和临时产物管理方式，减少后续维护成本。

## 一、目录职责（建议长期保持）

- `abap/`：ABAP 源码与对象清单（`objects.manifest.json`）
- `scripts/`：可执行自动化脚本（连接、读写、激活、批量推拉、健康检查）
- `scripts/lib/`：脚本共享库（当前为 `erpl-adt.ps1`）
- `.cursor/skills/`：项目级 AI skill 定义与参考资源
- `docs/`：流程、排障、规范、变更记录
- `.vscode/`：工作区设置与扩展推荐

## 二、脚本入口分层（建议）

### 2.1 核心单对象操作
- `sap-adt-read-object.ps1`
- `sap-adt-write-object.ps1`
- `sap-adt-activate.ps1`

### 2.2 批量操作
- `sap-adt-pull-from-manifest.ps1`
- `sap-adt-push-objects.ps1`
- `sap-adt-push-all.ps1`

### 2.3 质量与验证
- `sap-adt-healthcheck.ps1`
- `sap-adt-sidefx-class-cycle.ps1`

### 2.4 工具安装与整合
- `install-vscode-extensions.ps1`
- `install-marketplace-vsix.ps1`
- `fetch-abap-skills-assets.ps1`

## 三、临时文件管理规则

已通过 `.gitignore` 屏蔽常见临时产物：
- `tmp_*.log/txt/json`
- `abap/*.verify.*`、`abap/*.sidefx.*` 等验证快照
- `__pycache__/`、`*.pyc`

建议使用清理脚本：

```powershell
.\scripts\cleanup-project-temp.ps1 -WhatIf
.\scripts\cleanup-project-temp.ps1 -IncludeAbapSnapshots
```

## 四、命名规范（建议）

- 文档：`docs/<主题>-<目的>.md`（中英文均可，但建议统一风格）
- 脚本：`scripts/sap-adt-<动作>-<对象>.ps1`
- 一次性验证快照：放 `abap/`，并在结束后通过清理脚本移除

## 五、文档保留规则（强约束）

- 仅保留“已落地实现 + 可执行验证”的文档。
- 纯设想、路线图、愿景类文档（无对应脚本/命令/流程落地）默认不保留。
- 若需临时记录想法，应放到 issue 或任务系统，不进入 `docs/` 主干。
- 文档若引用脚本，必须包含可直接运行的命令示例。

## 六、后续可继续优化项

1. 将 `docs/` 按主题拆分子目录（`connection/`、`workflow/`、`troubleshooting/`）
2. 增加根目录 `README.md` 作为统一导航入口
3. 给批量推送脚本加 `-WhatIf` 与结构化输出（JSON）模式，便于 CI 集成
